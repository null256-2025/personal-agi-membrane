# Personal AGI Membrane 提出用資料

## AGIが社会へ出る前に、人間の境界を作る

---

## 1. 概要

Personal AGI Membrane は、AIエージェントが人間の代わりに外部と接触する時代のための、同意・権限・情報境界インターフェースです。

個人の生活、発信、仕事を `Private / Public / Business` の3つのAGI膜に分け、外部から来る依頼をそのまま実行するのではなく、現実に反映される直前で人間が `Yes / No / 追加指示` を判断できるようにします。

一文で言うと、これはAIチャットアプリではなく、AIが社会へ接触する前の Reality Gate です。

---

## 2. 背景にある課題

生成AIが進化すると、AIは質問に答えるだけではなく、人間の代わりに予定調整、登壇依頼、紹介、SNS投稿、商談前確認などを行うようになります。

しかし、AIエージェントが外部と接触し始めると、次の問題が発生します。

* 個人予定のどこまでを仕事相手に共有してよいか分からない
* 登壇や商談の条件確認をAIに任せたいが、最終承認なしに現実が進むのは危険
* SNS投稿やイベント参加報告で、仕事情報、公開情報、個人情報の境界が曖昧になる
* AI同士が何を参照し、何を渡し、何を止めたのか後から確認できない
* 自動化が進むほど、人間がどこで止めるべきかが見えにくくなる

既存のAIチャットや業務自動化ツールは、「何を生成するか」には強い一方、「何を現実に反映してよいか」「どの情報を誰に渡してよいか」「誰が最後に承認したか」を扱いきれていません。

---

## 3. 解決策

Personal AGI Membrane は、個人の社会的接触を3つのAGI膜で扱います。

| 領域 | 役割 | 扱う情報 |
| --- | --- | --- |
| Private AGI | 生活、予定、友人、家族、個人的事情を守る | 個人予定、私的事情、非公開の関係性 |
| Public AGI | 発信、イベント、紹介、コミュニティ上の自分を扱う | SNS、公開プロフィール、イベント参加、紹介 |
| Business AGI | 仕事、商談、登壇、見積、契約前確認を扱う | 依頼条件、公開可能な実績、業務判断 |

3つの膜は完全に分断されているわけではありません。原則は分離し、必要な時だけ限定的に情報を渡します。

例として、B社からAさんへ登壇依頼が届いた場合、Business AGI は条件や謝礼を確認します。Private AGI は空き時間だけを返し、予定名や前後の私的予定は渡しません。Public AGI は肩書き公開、SNS投稿、写真利用の可否を確認します。

最後に、3つの判断を Reality Gate に集約し、人間が現実に通すかどうかを決めます。

---

## 4. コア体験

### Reality Gate

人間に長いAI会話ログを読ませるのではなく、現実に反映される直前の判断だけを出します。

登壇依頼の場合、表示する情報は以下に絞ります。

* 依頼種別
* 依頼元
* 推奨判断
* 日時
* 場所
* 謝礼
* 共有OK
* 共有NG

### AGI Membrane Indicator

画面上部には、Private / Public / Business の3つの膜を表示します。

これは装飾ではなく、「今回の依頼がどの領域に触れているか」を示すインジケーターです。登壇依頼では Business が最も明るく、Public が中程度、Private は空き時間のみ参照するため控えめに表示されます。

### Agent Trace

通常時は表示しません。不安な時だけ長押しで開きます。

Trace では、以下を短く確認できます。

* 何を参照したか
* 何を渡したか
* 何を渡さなかったか
* なぜ人間承認が必要になったか

## 5. デモ内容

今回のプロトタイプでは、AさんとBさんのスマホ画面を横に並べ、AI同士の社会的接触と人間承認の流れを見せます。

### 登壇依頼シナリオ

B社のBusiness AGIが、Aさんへ登壇依頼を送ります。

```text
日時: 2026/07/01 15:00-16:00
場所: 京都市内
謝礼: 30万円
確認したいこと: 登壇可否、公開可能な肩書き、SNS投稿可否
```

Aさん側では、Private / Public / Business の3つの膜が依頼を判定し、Reality Gate に集約します。

### 操作

ユーザー操作は3つに絞ります。

* 右スワイプ: Yes / 通す
* 左スワイプ: No / 止める
* 下スワイプ: 音声で追加条件を伝える

明確な承認ボタンや管理画面風のメニューは置きません。現実に介入する直前の判断を、スワイプだけで成立させます。

### 自動デモ

登壇中に操作説明で止まらないよう、`AUTO DEMO` ボタンで以下を自動実行します。

1. Aさんが登壇依頼を右スワイプし、Yesとして承認する
2. Bさん側に、Aさんからの条件付き承認が届く
3. 次にAさんが同じ依頼を左スワイプし、Noとして止める
4. Bさん側に、Aさんが今回は見送ったことが届く
5. 最後にAさんが下スワイプし、音声で追加条件を伝える
6. 発話内容が3つの膜の上にタイプライター風に表示される
7. Bさん側に、Aさんからの追加条件が届く

このデモで見せたいのは、AさんがAIに命令している画面ではありません。BさんのAIから来た依頼が、AさんのPersonal AGI Membraneを通り、現実に返る直前で人間の同意を取る流れです。

## 6. UI方針

このプロトタイプは、SaaSダッシュボード、管理画面、チャットUIには見せません。

目指す見た目は、黒に近い深い背景、薄い光、余白、最小限の文字で構成された、近未来的で静かな承認体験です。

UIの主役はボタンではなく、Reality Gate の判断です。

避けるもの:

* チャットUI
* 管理画面風の一覧
* 明確な承認ボタン
* 情報量の多い表
* 過剰なネオン
* 白背景の業務アプリ風UI

目指すもの:

* Apple的な静けさ
* 膜のようなPrivate / Public / Businessの表現
* AGIが裏側で判断している気配
* 人間が最後に同意するための短い操作

## 7. 技術構成

現在のMVPは、スマホ向けWebアプリとして実装します。

### 実装方針

* Next.js
* TypeScript
* Tailwind CSS
* OpenAI APIはサーバー側Route Handlerから呼び出す
* API失敗時は固定フォールバックデータでデモ継続
* DB、認証、本番OAuth連携はMVPでは行わない

### AIの使い方

OpenAI APIは、依頼文と文脈を読み、以下を構造化JSONとして返すために使います。

* 依頼が触れる領域の分類
* Private / Public / Business ごとの判断
* 共有してよい情報
* 共有してはいけない情報
* Reality Gate に表示する短い判断文
* Agent Trace 用の説明

下スワイプで音声指示が追加された場合は、既存の判断に追加条件を反映し、Reality Gate を再生成します。

## 8. MVPでやること / やらないこと

### やること

* 登壇依頼シナリオの通しデモ
* Aさん/Bさんの並列画面
* Private / Public / Business の膜表示
* Reality Gate による最終判断
* 右スワイプ、左スワイプ、下スワイプ
* 音声入力の文字起こし演出
* Agent Trace のオーバーレイ表示
* API失敗時のフォールバック

### やらないこと

* 本物のスマートグラス連携
* 実カレンダー登録
* 実メール送信
* 実SNS投稿
* 決済
* 契約締結
* 完全自律エージェントとしての本番運用

このMVPの目的は、完全自動化ではなく、AIが社会へ接触する時の境界、同意、監査、人間承認の価値を検証することです。

---

## 9. 差別化

Personal AGI Membrane は、AIチャットアプリでも業務自動化ツールでもありません。

差別化は、AIが自律的に動くことを前提に、どこまで任せ、どこで止め、何を渡し、何を渡さなかったかを扱う点にあります。

主な差別化ポイント:

* AIチャットではなく、現実変更前の同意インターフェースである
* Private / Public / Business の3層で個人の境界を守る
* 自動化の速さではなく、止める場所を設計している
* AI同士のやり取りを普段は隠し、不安な時だけ Trace で確認できる
* スマホMVPから始め、将来的にはスマートグラス上のAmbient AGIへ拡張する

## 11. 5年後の姿

5年後、個人はスマホや眼鏡型デバイス上に、自分専用のPersonal AGI Membraneを持つようになります。

人間が誰かと会う前に、AGI同士が一次接触します。企業と商談する前に、Business AGI同士が前提条件をそろえます。イベント参加後にはPublic AGIが投稿候補を作り、予定や私的事情はPrivate AGIが守ります。

社会には、人間同士が直接会話する前の、AGI同士の接触層が生まれます。

その時に必要なのは、AIをより賢く見せるUIではなく、人間の境界を守りながら、現実を変える直前だけ同意を求めるインターフェースです。

---

## 12. 締め

Personal AGI Membrane は、AIが人間の代わりに社会へ出ていく時代に、人間のPrivate、Public、Businessの境界を守るためのプロダクトです。

AIが整理し、人間が最後にスワイプで同意する。

```text
Reality changes only when you allow it.
```
